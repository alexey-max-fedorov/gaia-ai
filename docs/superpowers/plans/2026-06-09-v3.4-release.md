# GAIA Code v3.4 Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship GAIA Code v3.4 — a skill-engine security guardrail, a third permission mode (Accept Edits), `/status` & `/help` built-ins, memory export + compaction, plan archiving, a richer `/update`, two new bundled skills (`/doctor`, `/pr-review`), CI version-sync enforcement, and a website changelog page.

**Architecture:** Three layers, edited in order: (1) `prompts/` — the product; prose specs read by an LLM inside Perplexity, so wording precision is the whole game and consistency across the four files is mandatory (the gate must describe what the engines define). (2) Repo docs — `CLAUDE.md` + `README.md`. (3) `website/` — Next.js 15 App Router site whose single source of truth is `website/lib/site.ts`. CI lands last so the version-sync check passes on its first run.

**Tech Stack:** Markdown prompt files (no build/test — verification is grep + careful re-read), Next.js 15 / React 19 / Tailwind / framer-motion (verified by `pnpm lint` + `pnpm build`), Node 22 ESM script + GitHub Actions for CI.

## Context for a zero-context engineer

- This repo is **not a runnable app**. `prompts/*.md` are deployed by hand into a Perplexity Space: `SYSTEM_INSTRUCTIONS.md` ("the gate") is pasted into the Space Instructions field; `SYSTEM_PROMPT.md`, `MEMORY_ENGINE.md`, `TURN_ENGINE.md` are uploaded as Space files. Skills (`update.md`, and the new `doctor.md`/`pr-review.md`) are optional uploads invoked as `/update` etc.
- "GAIA" referenced inside the prompt files is the LLM agent running in Perplexity — it has tools `search_web`, `fetch_url`, `execute_code` (Python sandbox, no internet), and GitHub MCP. It is **not** this repo's tooling.
- The version string lives in **four files** (README header, `website/package.json`, `website/lib/site.ts` `VERSION`, and `prompts/SYSTEM_PROMPT.md` in two spots). This release moves all of them from `3.3` to `3.4` and adds a CI check so they can never drift again.
- Website verification gates: `cd website && pnpm lint && pnpm build`. There is no test suite.
- Every commit message in this plan ends with the trailer paragraph `Co-Authored-By: Claude Fable 5 <noreply@anthropic.com>` (passed as a second `-m`).

**Explicitly out of scope (decided during brainstorming):** a `/commit-style` skill (commit style is already enforced by `SYSTEM_PROMPT.md` PART IV), a `/changelog` skill (covered by Task 6's `/update` upgrade and the website changelog page), automatic GitHub sync of `MEMORY.md` (export/import covers the need), a CI markdown link-checker (external-link checks flake), and reformatting historical rows in the README version table (`3.0.1` vs `3.1` are the version strings actually released — rewriting history would break `/update`'s comparison against real deployed Spaces).

   - `add_comment_to_pending_review` for each line-anchored finding,
   - `pull_request_review_write` method `submit_pending` with event `COMMENT`.

   Submit as **`COMMENT` only** — never `APPROVE` or `REQUEST_CHANGES` unless the
   user explicitly asks for that event.

## Notes

  step 4.
- Never include code suggestions you have not verified against the actual file

```js
#!/usr/bin/env node
// scripts/check-version-sync.mjs
// The release version lives in four files; fail when any of them drift.
// Run from anywhere: node scripts/check-version-sync.mjs
import { readFileSync } from "node:fs";

const read = (p) => readFileSync(new URL(`../${p}`, import.meta.url), "utf8");

const found = {};

found["README.md header"] = read("README.md").match(/^\*\*Version:\*\* (\S+)/m)?.[1];
found["website/package.json"] = JSON.parse(read("website/package.json")).version;
found["website/lib/site.ts VERSION"] = read("website/lib/site.ts").match(
  /export const VERSION = "([^"]+)"/,
)?.[1];

const prompt = read("prompts/SYSTEM_PROMPT.md");
found["SYSTEM_PROMPT.md header"] = prompt.match(/SYSTEM PROMPT \(v([\d.]+)\)/)?.[1];
const footers = [...prompt.matchAll(/Running GAIA Code ([\d.]+)/g)].map((m) => m[1]);
footers.forEach((v, i) => {
  found[`SYSTEM_PROMPT.md footer #${i + 1}`] = v;
});
if (footers.length === 0) found["SYSTEM_PROMPT.md footer"] = undefined;

for (const [where, v] of Object.entries(found)) {
  console.log(`${(v ?? "NOT FOUND").padEnd(10)} ${where}`);
}

const versions = new Set(Object.values(found));
if (versions.size !== 1 || versions.has(undefined)) {
  console.error("\n✗ Version drift detected — all locations must carry the same version.");
  process.exit(1);
}
console.log(`\n✓ All version strings agree: ${[...versions][0]}`);
```
